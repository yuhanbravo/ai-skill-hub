# Project Lifecycle

这个文档定义 skill-hub 的项目生命周期模型，用来说明不同 skill 在协作链路中的位置。

## Lifecycle Pattern

`handoff -> takeover -> status -> audit -> governance`

## Stages

- `handoff`：任务交接阶段。重点是把目标、边界和预期输出交给下游执行方。
- `takeover`：首次理解阶段。重点是进入陌生项目并建立最小可执行认知。
- `status`：持续状态更新阶段。重点是把近期进展、风险和待办整理成可同步输出。
- `audit`：规则检查阶段。重点是检查项目是否符合结构、规则或约束，并形成报告。
- `governance`：结构与质量治理阶段。重点是基于前面阶段的输入持续改进项目文档、结构和协作质量。

## Relationship

- `handoff` 解决“任务如何交给别人做”。
- `takeover` 解决“第一次进入项目如何理解它”。
- `status` 解决“项目当前处于什么状态”。
- `audit` 解决“项目是否符合既定规则”。
- `governance` 解决“如何把这些输入沉淀为长期稳定的治理机制”。

这些阶段不是互斥模块，而是一条可循环的协作链路：任务先被 handoff，项目被 takeover，状态被持续更新，规则通过 audit 被检查，最终进入 governance 并反过来提升前面各阶段的质量。
