# 接管前快速核对清单

在开始实施前，快速确认以下事项：

- 我是否已经拿到一份明确的任务包
- 任务包里是否写清了目标、范围、边界和验收标准
- 目标文件或目录是否明确
- 是否已有项目本地规则需要优先遵守
- 是否需要先输出对任务的理解，再开始施工
- 本次是否要求回传 execution report
- 如果发现额外问题，是否已约定只记录不扩做
- 如果本轮要更新项目 handoff 文档，是否优先检查并更新 `docs/HANDOFF.md`
- 是否已确认优先更新这些 section：`Update Log`、`Current Status/当前状态`、`Hard Boundaries/边界`、`Recommended Next Steps/下一步建议`、`Environment Blockers/环境阻塞`
- 如果 `docs/HANDOFF.md` 里没有 `## Update Log`，是否准备补建而不是重写全文件
- 如果仓库里已有 `minimal_handoff_manual.md` 或其他历史 handoff 生成物，是否已经明确它们只作为 legacy/generated 保留
- execution report 是否会明确写出：改了什么、没改什么、如何验证、当前阻塞、下一步建议

如果以上有关键项缺失，先做最小假设，并在 execution report 中写明。
