# REVIEW_PACKET

## 使用边界

- `task package`：给实施侧开工前使用，强调目标、范围、约束、验收
- `execution report`：给实施完成后回传实际改动、验证、阻塞和边界遵守情况
- `review packet`：给 ChatGPT / 上游审阅侧使用，强调“本轮值得审的增量摘要”和“需要重点审什么”

说明：

- `review packet` 不重新定义任务
- `review packet` 不替代 `execution report`
- 它的主要作用是帮助快速审阅，减少复制粘贴
