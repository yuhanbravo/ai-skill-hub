# Naming Rules

This file explains naming expectations already defined by `SKILL.md` and machine-enforced by `default_governance.json`.

## Preferred Style

Use lowercase kebab-case where practical.

Examples:

- `system-architecture.md`
- `trade-flow-design.md`
- `decision-log.md`

## Forbidden Suffixes

Examples of names that should be rejected by governance checks:

- `*_v2.md`
- `*_final.md`
- `*_updated.md`

## Canonical Naming Hint

When a readable derivative exists, relate it by topic rather than by version suffix.

Examples:

- `docs/architecture/system-architecture.md`
- `docs_readable/architecture/system-architecture.md`
